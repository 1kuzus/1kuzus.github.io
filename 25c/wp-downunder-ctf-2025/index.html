check2:

也就是password[5]="p", pasword[6]="h"

check3:

def check_three(password):

def check_ex(password, func):
    GetIntCallbackFn = ctypes.CFUNCTYPE(ctypes.c_int, ctypes.c_wchar_p)

    class CallbackTable(ctypes.Structure):
        _fields_ = [("E", GetIntCallbackFn)]

    @GetIntCallbackFn
    def eval_int(v):
        print(func, ":", v)
        return int(eval(v))

    table = CallbackTable(E=eval_int)
    helper = get_helper()
    helper[func].argtypes = [ctypes.POINTER(CallbackTable)]
    helper[func].restype = ctypes.c_int
    return helper[func](ctypes.byref(table))

check4:

check4同样会调用eval_int函数，可以从打印出的参数得知check4是在对密码中剩下的位做校验。

静态分析比较复杂，但是注意到在check4的反汇编中，如果检查的某几个位置不符合规则，则函数会提前返回；因此可以根据eval_int函数的被调用次数大致判断check4的返回点。每次打印出的eval_int参数就是当前在检查的某几个位。考虑爆破：

Rev/SwiftPasswordManager: LoadMe

保存功能会根据用户输入的一个密码派生出AES加密密钥，然后对数据加密保存；本题需要我们逆向保存功能的逻辑，实现加载功能。附件已经给出了输出数据passwords.spm和密码DUCTF2025!。

输出数据中主要有随机数salt、nonce、tag和密文，下面的hook.js主要hook了三个函数：

AES.GCM.Nonce.init()，可以获得nonce值从而得到其在输出数据中的偏移；

swift_stdlib_random，同理这是为了确认salt的偏移；

SHA256.update(bufferPointer:)，密钥派生过程。

JavaScript
let iter_count = 0

Module.enumerateSymbolsSync("swiftpasswordmanager").forEach(function (sym) {
    // AES.GCM.Nonce.init()

        console.log(sym.name, sym.address);
        Interceptor.attach(sym.address, {
            onLeave: function (retval) {
                const rax = this.context.rax;
                const rdx = this.context.rdx;
                console.log(rax, rdx);
            }
        });
    }
    // SHA256.update(bufferPointer:)
    else if (sym.name.indexOf("$s6Crypto6SHA256V6update13bufferPointerySW_tF") !== -1) {
        console.log(sym.name, sym.address);
        Interceptor.attach(sym.address, {
            onEnter: function (args) {
                iter_count++;
                if (iter_count < 4) {
                    const start = args[0];
                    const end = args[1];
                    const size = end.sub(start).toInt32();
                    const buffer = Memory.readByteArray(start, size);
                    console.log("iter_" + iter_count + ":");
                    console.log("Buffer content (hex):", hexdump(buffer, {length: size}));
                }
            },
        });
    }
});

利用动态分析的信息，能更容易（相比于读反汇编代码）得到：

1.

输出数据的结构：

Plain Text

header........  ls  salt......
..............................
..................  ln  nonce.
..............  lt  tag.......
..................  lc....  ct

其中ls、ln、lt、lc分别为salt、nonce、tag和密文的长度；大小分别为2B、2B、2B和4B。

2.

密钥派生过程：key=sha256(key + salt)迭代43690次。

例如输出的iter_1，sha256的参数以原始密码开头；所有iter中sha256的参数都以salt值结尾。

最后的解题脚本：

Python
import hashlib
from Crypto.Cipher import AES

with open("passwords.spm", "rb") as f:
    encrypted_data = f.read()

salt = encrypted_data[10:42]
nonce = encrypted_data[44:56]
tag = encrypted_data[58:74]
ciphertext = encrypted_data[78:]

print("salt:", salt.hex())
print("nonce:", nonce.hex())
print("tag:", tag.hex())
print("ciphertext:", ciphertext.hex())

password = "DUCTF2025!"
key = password.encode("utf-8")

for i in range(43690):
    key = hashlib.sha256(key + salt).digest()

cipher = AES.new(key, AES.MODE_GCM, nonce=nonce)
plaintext = cipher.decrypt_and_verify(ciphertext, tag)
print(plaintext)
